# SHL Assessment Recommender

A conversational AI agent that helps hiring managers find the right SHL assessments through dialogue.

## Architecture

```
User → POST /chat → FastAPI → LangGraph Agent → Groq LLM
                                    ↓
                              Chroma Vector Store (SHL Catalog)
```

### Agent Flow

```
classify_intent
    ├── off_topic  → refuse (polite refusal)
    ├── greeting   → greet (warm intro)
    ├── clarify    → extract_facts → generate_clarification
    ├── recommend  → extract_facts → retrieve → rerank → generate_reply
    ├── refine     → extract_facts → retrieve → rerank → generate_reply
    └── compare    → extract_facts → compare_assessments (catalog-grounded)
```

## Setup

### 1. Clone & Install

```bash
git clone <your-repo>
cd shl-recommender
pip install -r requirements.txt
```

### 2. Configure Environment

```bash
cp .env.example .env
# Edit .env and add your GROQ_API_KEY
```

Get a free Groq API key at: https://console.groq.com

### 3. Scrape the SHL Catalog

```bash
python -m scripts.scrape_catalog
```

This scrapes `https://www.shl.com/solutions/products/product-catalog/` (Individual Test Solutions only) and saves to `data/catalog.json`.

### 4. Build Vector Index

```bash
python -m scripts.build_index
```

Embeds all assessments using `all-MiniLM-L6-v2` and stores in Chroma at `data/chroma/`.

### 5. Run the Server

```bash
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
```

### 6. Test

```bash
# Health check
curl http://localhost:8000/health

# Chat
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "user", "content": "I am hiring a mid-level Java developer"}
    ]
  }'
```

## API Reference

### GET /health
Returns `{"status": "ok"}` with HTTP 200.

### POST /chat
**Request:**
```json
{
  "messages": [
    {"role": "user", "content": "..."},
    {"role": "assistant", "content": "..."},
    {"role": "user", "content": "..."}
  ]
}
```

**Response:**
```json
{
  "reply": "Here are 5 assessments for a mid-level Java developer...",
  "recommendations": [
    {"name": "Java 8 (New)", "url": "https://www.shl.com/...", "test_type": "K"},
    {"name": "OPQ32r", "url": "https://www.shl.com/...", "test_type": "P"}
  ],
  "end_of_conversation": false
}
```

**Schema rules (non-negotiable):**
- `reply`: always present, non-empty string
- `recommendations`: empty `[]` when clarifying or off-topic; 1–10 items when recommending
- `end_of_conversation`: `true` only when agent has committed to final shortlist
- Every `url` must be from `shl.com`

## Design Decisions

### Retrieval
- **Sentence Transformers (all-MiniLM-L6-v2)**: Local embedding, no API cost, fast
- **Chroma**: Persistent, supports metadata filtering (remote, type)
- **Rich search text**: Each assessment is embedded as a paragraph (name + type + description + metadata), not just name
- **Two-stage**: Semantic retrieval (top 25) → LLM re-rank (top 10)

### Clarification Strategy
- Ask **ONE question per turn** — multi-question turns confuse users
- Priority order: job_role → what_to_measure → seniority → remote → specific_skills
- After 2-3 clarifying turns, recommend with available context (don't over-clarify)
- Force recommendation by turn 7/8 regardless

### Anti-Hallucination
- Assessment names and URLs are **always injected from Chroma metadata** — never generated by LLM
- LLM only writes the `reply` text
- Comparison answers explicitly prompted to use ONLY provided catalog data
- URL validation in API layer (shl.com check)

### Turn Budget
- Max 8 turns (spec requirement)
- Turn counter tracked in state
- Forced recommendation intent when approaching limit

## Stack

| Component | Choice | Reason |
|-----------|--------|--------|
| LLM | Groq (llama-3.3-70b) | Free, fast, high quality |
| Vector DB | Chroma | Open-source, persistent, metadata filters |
| Embeddings | all-MiniLM-L6-v2 | Local, fast, good quality |
| Agent Framework | LangGraph | Fine-grained control, explicit state |
| API | FastAPI | Fast, async, auto-docs |
| Scraping | requests + BS4 | Lightweight, SHL pages are SSR |
